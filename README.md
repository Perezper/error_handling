# error_handling

Este repositorio define el **estándar de manejo de errores** para los
microservicios de la empresa. Está diseñado para usarse como base en
un GitHub Copilot Workspace y complementar los repos `request_estandar`
y `response_estandar`.

Contiene:

- `docs/`: documentación de estrategias, catálogo de errores y lineamientos.
- `examples/`: ejemplos de errores HTTP y manejadores en código.
- `schemas/`: estructuras sugeridas para clasificación y registro de errores.

Cuando generes endpoints o manejadores de excepciones con GitHub Copilot,
indica que siga lo definido en `docs/MANEJO_ERRORES.md` y los demás documentos.
